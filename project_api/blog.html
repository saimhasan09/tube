Var:

  • Function Scope : Variables that have the var prefix are scoped to functions. They can only be used from within the defined function.
  • Hoisting : We can use variables declared with var before they are declared, but they will be initialized with undefined until their actual declaration. Variables declared with var are hoisted to the top of their containing function or global scope.
let:

  • Block Scope : Let-declared variables are block-scoped, which means that you can only access them within the block in which they are defined.
  • No Hoisting : Let-declared variables are also pushed to the top of the enclosed block; but, as they are not initialised, you cannot use them prior to their declaration.
Const:

  • Block Scope : const declared variables are block-scoped, which means that you can only access them within the block in which they are defined.
  • NULL : Null is frequently used as a signal of an empty state or to purposefully indicate the lack of any object value or no value.It can be used to a variable to indicate that a meaningful value is missing.
  • Undefined : The value produced by functions that do not explicitly return a value, or the undefined value, is the uninitialized, default value of variables.It is frequently used to determine whether a property exists or whether a variable has been defined.
  • REST API : An application programming interface, or REST API, complies with the requirements of the REST architectural style and facilitates communication with RESTful web offerings.
